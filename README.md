## 📌 ВВЕДЕНИЕ

Ariona OS v2.0 — это 16-битная операционная система, написанная на языке ассемблера NASM. Она загружается с дискеты (1.44 МБ) и работает на реальном железе или в эмуляторе QEMU.

### Что внутри:
- **7 приложений**
- **Собственный язык программирования** (Ariona Script Engine)
- **GitHub-клиент** (XOR-шифрование)
- **Управление железом** через порты
- **Файловая система** на дискете
- **Текстовый редактор**
- **Архиватор и распаковщик**

## 🚀 УСТАНОВКА И СБОРКА

### 1. Установить программы
**Windows:**  
Скачай [NASM](https://nasm.us), [QEMU](https://qemu.org), [Git](https://git-scm.com) и добавь в PATH.

**Linux:**
```bash
sudo apt update
sudo apt install nasm qemu-system-x86 git
```

**Mac:**
```bash
brew install nasm qemu git
```

### 2. Собрать ОС
**Вручную:**
```bash
nasm -f bin boot.asm -o boot.bin
nasm -f bin kernel.asm -o kernel.bin
copy /b boot.bin + kernel.bin ariona.img
```

**Или скриптом `build.bat`:**
```batch
@echo off
echo Building Ariona OS v2.0...
echo ====================================
echo Assembling bootloader...
nasm -f bin boot.asm -o boot.bin
if errorlevel 1 goto error
echo Assembling kernel...
nasm -f bin kernel.asm -o kernel.bin
if errorlevel 1 goto error
echo Creating disk image...
copy /b boot.bin + kernel.bin ariona.img
echo ====================================
echo SUCCESS! Created ariona.img
goto end
:error
echo ERROR! Build failed.
:end
pause
```

### 3. Запустить
**В QEMU:**
```bash
qemu-system-x86_64 -fda ariona.img -m 64 -vga std
```

**На реальном ПК:**
1. Запиши `ariona.img` на дискету (RawWrite или `dd`)
2. Вставь дискету, настрой BIOS (Boot Order → Floppy Drive первым)
3. Перезагрузи компьютер

## 🎮 ПРИЛОЖЕНИЯ

### 1. Ariona Script Engine (ASE)

Полноценный интерпретатор для управления графикой и состояниями. Работает в графическом режиме 320×200, 256 цветов.

**Команды ASE:**

| Клавиша | Команда | Что делает |
|---------|---------|------------|
| `1` | `FLAG` | Установить флаг состояния |
| `2` | `FLAG1` | Установить `print` + `flagdata` |
| `3` | `CNDT` | Установить условия (`cndt1`, `flagcnd`) |
| `4` | `CNDT` | То же, что и 3 |
| `5` | `PORT_DATA` | Проверить режим рисования и нарисовать пиксель |
| `6` | `FLAG2` | Установить `print` |
| `7` | `EXPORT` | Сохранить состояние на дискету (сектор 10) |
| `8` | `IMPORT` | Загрузить состояние с дискеты (сектор 10) |
| `@` | `IMPORT` | Включить режим рисования |
| `0` | `EXIT` | Выйти в главное меню |

**Переменные состояния:**
- `dov_x`, `dov_y` — координаты
- `dov_color` — цвет
- `dov_flags` — флаг состояния
- `dov_flagdata` — данные флага
- `dov_cndt1`, `dov_flagcnd` — условия
- `dov_eq` — режим рисования

**Пример работы:**
```
CMD> 1
Enter flag value: 1

CMD> 5
Pixel drawn!

CMD> 7
Exporting state to disk... State saved to sector 10!

CMD> 8
Importing state from disk... State loaded from sector 10!

CMD> 0
[Выход в меню]
```

### 2. Command Line

Терминал для работы с файлами.

**Команды:**

| Команда | Что делает |
|---------|------------|
| `D` | Показать список файлов на дискете |
| `f` | Создать файл |
| `d` | Удалить файл |
| `c` | Закрыть файл |
| `*` | Проверка |
| `p` | PS mode (переключить графику/текст) |
| `P` | Упаковать файлы в архив (сектор 60) |
| `U` | Распаковать архив (показать содержимое) |
| `X` | Выйти в главное меню |

**Подкоманды PS:**
| Клавиша | Что делает |
|---------|------------|
| `w` | Графический режим (320×200, 256 цветов) |
| `r` | Текстовый режим (80×25) |

---

### 3. Port I/O

Управление железом через порты.

**Команды:**

| Клавиша | Что делает | Порт | Значение |
|---------|------------|------|----------|
| `W` | Выключить клавиатуру | 0x64 | 0xAD |
| `A` | Включить клавиатуру | 0x64 | 0xAE |
| `F` | Включить звук | 0x61 | 0x03 |
| `Z` | Выключить звук | 0x61 | 0x00 |
| `k` | Сбросить CPU | 0x64 | 0xFE |
| `R` | Проверить флаги | — | — |
| `S` | Сохранить состояние на диск (сектор 30) | — | — |
| `L` | Загрузить состояние с диска (сектор 30) | — | — |
| `X` | Выйти в главное меню | — | — |

**Флаги состояния:**
| Флаг | Описание |
|------|----------|
| `key_flag` | W (клавиатура выключена) |
| `key_flag1` | A (клавиатура включена) |
| `key_flag2` | F (звук включен) |
| `key_flag3` | Z (звук выключен) |
| `key_flag4` | k (CPU сброшен) |

---

### 4. File Manager + GitHub

Управление файлами и эмуляция GitHub.

**Команды:**

| Клавиша | Что делает |
|---------|------------|
| `1` | Показать список файлов в репозитории |
| `2` | Загрузить файл (XOR-шифрование, сектор 40) |
| `3` | Скачать файл (XOR-расшифровка, сектор 40) |
| `4` | Установить GitHub токен |
| `5` | Выйти в главное меню |

**XOR-шифрование:**
- Ключ: 42
- Файлы шифруются побайтово: `b ^ 42`
- Зашифрованные файлы хранятся на дискете
- Расшифровка происходит автоматически при скачивании

### 5. Text Editor

Редактирование текстовых файлов.

**Команды:**

| Клавиша | Что делает |
|---------|------------|
| `1` | Создать новый файл (ввести текст) |
| `2` | Загрузить файл (показать текст из буфера) |
| `3` | Сохранить файл на дискету (сектор 50) |
| `4` | Загрузить на GitHub (XOR-шифрование на дискету) |
| `5` | Скачать с GitHub (XOR-расшифровка с дискеты) |
| `6` | Выйти в главное меню |

### 7. Ariona Unpacker

Распаковка архива из сектора 60 в сектора 70+.

**Что делает:**
- Читает архив с дискеты (сектор 60)
- Распаковывает файлы в сектора 70, 71, 72...
- Выводит имена файлов и секторы, куда они записаны
- Показывает количество распакованных файлов

**Пример работы:**
```
========================================
Application: Ariona Unpacker
========================================
Reading archive from disk (sector 60)...
Extracting: test1.txt -> sector 70
Extracting: test2.txt -> sector 71
Extracting: game.ari -> sector 72
Total files extracted: 3
All files extracted!
Press any key...
```

## 🔧 ТЕХНИЧЕСКИЕ ДЕТАЛИ

- **CPU:** 16-битный реальный режим (Real Mode), регистры AX, BX, CX, DX, SI, DI, SP, BP, CS, DS, ES, SS
- **Память:** 0x7C00 — загрузчик, 0x1000 — ядро, 0x9000 — стек, 0xA000 — VGA, 0xB800 — текст
- **Порты:** 0x60 (клавиатура), 0x64 (управление), 0x61 (звук), 0x70 (CMOS), 0x80 (отладка)
- **Прерывания:** INT 0x10 (видео), INT 0x13 (диск), INT 0x16 (клавиатура)
