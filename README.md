# ============================================================
# 1. ВВЕДЕНИЕ
# ============================================================

Ariona OS v2.0 — это 16-битная операционная система,
написанная на языке ассемблера NASM. Она загружается с дискеты
(1.44 МБ) и работает на реальном железе или в эмуляторе QEMU.

## Что внутри:
- 6 приложений
- Собственный язык программирования (Ariona Script Engine)
- GitHub-клиент (XOR-шифрование)
- Управление железом через порты
- Файловая система на дискете
- Текстовый редактор

# ============================================================
# 2. СИСТЕМНЫЕ ТРЕБОВАНИЯ
# ============================================================

## ДЛЯ РАЗРАБОТКИ:
- ОС: Windows / Linux / Mac
- NASM 2.15+ (сборка кода)
- QEMU 6.0+ (запуск в эмуляторе)
- Git 2.30+ (работа с GitHub)

## ДЛЯ ЗАПУСКА В ЭМУЛЯТОРЕ:
- QEMU 6.0+
- Память: 64 МБ (можно изменить)
- Видео: VGA (std)

## ДЛЯ ЗАПУСКА НА РЕАЛЬНОМ ПК:
- Процессор: Intel 80386 (i386) или выше
- ОЗУ: 1 МБ (рекомендуется 8 МБ)
- Дисковод: 3.5" FDD (1.44 МБ)
- Видео: VGA или совместимое
- Клавиатура: PS/2 или AT
- BIOS: Совместимый с IBM PC

# ============================================================
# 3. УСТАНОВКА И СБОРКА
# ============================================================

## 3.1. УСТАНОВИТЬ ПРОГРАММЫ

### Windows:
1. Скачай NASM с nasm.us
2. Скачай QEMU с qemu.org
3. Скачай Git с git-scm.com
4. Добавь в PATH (переменные среды)

### Linux:
```bash
sudo apt update
sudo apt install nasm qemu-system-x86 git
```

### Mac:
```bash
brew install nasm qemu git
```

## 3.2. СОБРАТЬ ОС

### Вручную:
```bash
nasm -f bin boot.asm -o boot.bin
nasm -f bin kernel.asm -o kernel.bin
copy /b boot.bin + kernel.bin ariona.img
```

### Скриптом (build.bat):
```batch
@echo off
echo Building Ariona OS v2.0...
echo ====================================
echo Assembling bootloader...
nasm -f bin boot.asm -o boot.bin
if errorlevel 1 goto error
echo Assembling kernel...
nasm -f bin kernel.asm -o kernel.bin
if errorlevel 1 goto error
echo Creating disk image...
copy /b boot.bin + kernel.bin ariona.img
echo ====================================
echo SUCCESS! Created ariona.img
goto end
:error
echo ERROR! Build failed.
:end
pause
```

## 3.3. ЗАПУСТИТЬ

### В QEMU:
```bash
qemu-system-x86_64 -fda ariona.img -m 64 -vga std
```

### С отладкой:
```bash
qemu-system-x86_64 -fda ariona.img -s -S
# Подключить GDB: target remote localhost:1234
```

### На реальном ПК:
1. Запиши ariona.img на дискету (программой RawWrite или dd)
2. Вставь дискету в дисковод
3. Настрой BIOS: Boot Order → Floppy Drive первым
4. Перезагрузи компьютер

### Запись на дискету:
```bash
# Linux:
sudo dd if=ariona.img of=/dev/fd0 bs=512 count=2880

# Windows (с dd):
dd if=ariona.img of=\\.\a: bs=512
```

# 5. ПРИЛОЖЕНИЕ 1: ARIONA SCRIPT ENGINE
# ============================================================

Это полноценный интерпретатор для управления графикой
и состояниями. Работает в графическом режиме 320x200, 256 цветов.

## 5.1. КОМАНДЫ:

| Клавиша | Команда | Что делает |
|---------|---------|------------|
| 1 | Set flag | Установить флаг состояния |
| 2 | Set flag1 | Установить print и flagdata |
| 3 | Set condition | Установить условие (cndt1, flagcnd) |
| 4 | Set driver | Установить цвет (driver) |
| 5 | Paint pixel | Нарисовать пиксель в (X,Y) цветом Color |
| 6 | Set flag2 | Установить print для flag2 |
| 7 | Export MPL | Сохранить состояние на дискету (сектор 10) |
| 8 | Import MPL | Загрузить состояние с дискеты (сектор 10) |
| @ | Library | Режим библиотеки (линии и круги) |
| 0 | Exit | Выйти в главное меню |

## 5.2. ПЕРЕМЕННЫЕ (СОСТОЯНИЕ):

| Переменная | Описание |
|------------|----------|
| X | Координата X (0-319) |

| Color | Цвет (0-255) |
| Flags | Флаг состояния |
| Flagdata | Данные флага |
| Cndt1 | Условие 1 |
| Flagcnd | Флаг условия |
| Map | Режим рисования |

## 8.2. XOR-ШИФРОВАНИЕ:

- Ключ: 42
- Файлы шифруются побайтово: `b ^ 42`
- Зашифрованные файлы хранятся на дискете
- Расшифровка происходит автоматически при скачивании

## 8.3. КАК ПОЛУЧИТЬ GITHUB ТОКЕН:

1. Зайди на github.com
2. Settings → Developer settings
3. Personal access tokens → Tokens (classic)
4. Generate new token (classic)
5. Выбери права: repo, workflow, write:packages
6. Нажми Generate token
7. Скопируй токен!

# 12. РАБОТА С ДИСКЕТОЙ
# ============================================================

## 12.1. ЧТЕНИЕ С ДИСКЕТЫ (INT 0x13):

```nasm
mov ah, 0x02          ; Чтение
mov al, 1             ; 1 сектор
mov ch, 0             ; Дорожка 0
mov cl, 10            ; Сектор 10
mov dh, 0             ; Головка 0
mov dl, 0x00          ; Диск A:
mov bx, buffer        ; Буфер
int 0x13
```

## 12.2. ЗАПИСЬ НА ДИСКЕТУ (INT 0x13):

```nasm
mov ah, 0x03          ; Запись
mov al, 1             ; 1 сектор
mov ch, 0             ; Дорожка 0
mov cl, 10            ; Сектор 10
mov dh, 0             ; Головка 0
mov dl, 0x00          ; Диск A:
mov bx, buffer        ; Данные
int 0x13
```

# 16. ЧАСТЫЕ ВОПРОСЫ (FAQ)
# ============================================================

## Q: Можно ли запустить на реальном компьютере?
A: Да! Нужен компьютер с дисководом 3.5" и процессором 80386+.

## Q: Сколько памяти нужно?
A: Минимум 1 МБ, рекомендуется 8 МБ.

## Q: Почему не работает клавиатура?
A: В приложении Port I/O клавиатура может быть выключена. Нажми `A` чтобы включить.

## Q: Как выйти из приложения?
A: Нажми `0` или `X` (зависит от приложения).

## Q: Можно ли сохранять файлы?
A: Да! Все приложения умеют сохранять на дискету.

## Q: Работает ли реальная сеть?
A: Нет, работа с GitHub — это эмуляция через дискету.

## Q: Как добавить свои программы?
A: Добавь новый `app_` в kernel.asm и пункт в меню.

## Q: Можно ли использовать мышь?
A: Нет, только клавиатура.

## Q: Какие форматы файлов поддерживаются?
A: Только текстовые файлы.

## Q: Почему приложение зависло?
A: Возможно, вы выполнили команду перезагрузки (k) или выключения.

## Q: Как перезагрузить Ariona OS?
A: Нажми Ctrl+Alt+Delete в QEMU или перезагрузи компьютер.

## 17.1. BOOT.ASM — ЗАГРУЗЧИК

```nasm
; ============================================
; ЗАГРУЗЧИК ARIONA OS
; ============================================
org 0x7C00
bits 16

start:
    cli
    xor ax, ax
    mov ds, ax
    mov es, ax
    mov ss, ax
    mov sp, 0x7C00
    sti

    mov ax, 0x0003
    int 0x10

    mov si, msg_loading
    call print

    mov ah, 0x02
    mov al, 32
    mov ch, 0
    mov cl, 2
    mov dh, 0
    mov dl, 0x00
    mov bx, 0x1000
    int 0x13
    jc error

    jmp 0x1000:0x0000

error:
    mov si, msg_error
    call print
    jmp $

print:
    lodsb
    or al, al
    jz .done
    mov ah, 0x0E
    int 0x10
    jmp print
.done:
    ret

msg_loading db 'Loading Ariona OS...', 0x0D, 0x0A, 0
msg_error db 'Error loading kernel!', 0

times 510-($-$$) db 0
dw 0xAA55
```
